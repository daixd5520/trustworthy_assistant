# 食物热量估算

这个技能用于分析食物照片，识别餐盘中的主要食物，并给出一个保守的热量估算区间。

这是一版轻量实现：

- 优先使用 `read_image`
- 不依赖外部营养数据库
- 不承诺精确克数
- 默认输出“估算值 + 不确定性”

## 什么时候用

- 用户发送了饭菜、零食、饮料、外卖、餐盘、菜单照片
- 用户问“这顿多少卡”“帮我看看热量”“这碗面大概多少千卡”
- 用户想知道蛋白质、碳水、脂肪的大概构成
- 用户想减脂、控卡，问这张图里的食物是否适合

## 优先工具

### `read_image`

收到食物图片后，优先调用 `read_image`，并使用类似下面的分析目标：

1. 识别图片里可能的食物种类
2. 判断每种食物的大致份量
3. 识别是否有高油、高糖、高热量成分
4. 给出整餐热量区间
5. 如果图片不足以判断，明确说不确定点

推荐提示词风格：

```text
请识别这张食物照片中的主要食物、可能份量、烹饪方式和高热量来源。
然后估算每个组成部分的大致热量，并给出整餐总热量区间。
如果无法准确判断，请明确说明不确定性，并给出保守估算。
```

## 回答原则

- 默认使用“区间估算”，例如 `450-600 kcal`
- 不要伪装成精确称重结果
- 如果是奶茶、炸物、火锅、盖浇饭、拌饭、重油菜，主动提高不确定性提示
- 如果看不清份量，就说明“图片无法准确判断重量”
- 如果是包装食品且营养标签清晰，优先参考标签，而不是瞎估

## 建议输出格式

优先按这个结构回答：

- 识别结果：这张图里大概是什么
- 热量估算：总热量区间
- 主要来源：哪几部分最占热量
- 营养判断：大致蛋白质/碳水/脂肪结构
- 不确定性：哪些地方看不清、可能误差较大
- 建议：如果用户在控卡，给一个简短建议

## 简单估算经验

当图片信息有限时，可参考这些常见经验值做保守估计：

- 一小碗米饭：约 `180-230 kcal`
- 一拳头熟面：约 `220-320 kcal`
- 一份煎炸鸡腿/鸡排：约 `250-450 kcal`
- 一杯含糖奶茶：约 `250-500 kcal`
- 一勺明显的辣油/汤油：可能额外增加 `80-150 kcal`
- 一份炒饭/盖饭：常见在 `500-900 kcal`
- 一份沙拉：跨度很大，取决于酱料、坚果、肉类和分量

这些只是兜底经验，优先依据图片内容判断，不要生搬硬套。

## 示例

用户：“这顿饭多少卡？”

处理：

- 调用 `read_image`
- 识别米饭、鸡肉、炒青菜、汤汁或油脂
- 输出热量区间和主要来源

用户：“我在减脂，这个能吃吗？”

处理：

- 先给热量估算
- 再说是否适合作为减脂餐
- 如果油脂或主食过多，明确指出

用户：“这个奶茶大概多少热量？”

处理：

- 结合杯型、是否奶盖、是否珍珠、是否全糖做估算
- 明确说明这是按常见门店配方估算，不是精确值
